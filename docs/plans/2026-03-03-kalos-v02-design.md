# Kalos v0.2.0 Design — Tailwind Adapter + Brand Template + Multi-Brand

**Date:** 2026-03-03
**Status:** Approved
**Approach:** C — Combined but staged (schema designed for multi-brand, Tailwind adapter first, then brand layer on top)

---

## Problem

Kalos v0.1.0 only validates and syncs to Pencil. Real projects also use Tailwind CSS, and corporate projects need multi-brand support (multiple color palettes and typefaces under one config). Without these, Kalos only governs half the design surface.

## Solution

Add Tailwind adapter (sync + validate), a brand template with strict enforcement, multi-brand palette support in the config schema, an extract command to bootstrap config from existing projects, and brand-aware behavior across all subcommands.

---

## Config Schema Changes

### Multi-Brand Support

New `brands:` top-level key. Optional — single-brand configs work without it.

```yaml
# .kalos.yaml — with multi-brand
extends: brand
version: 0.2.0

tokens:
  colors:
    primary: "#6366F1"
    secondary: "#EC4899"
    neutral: "zinc"
    semantic:
      success: "#22C55E"
      warning: "#F59E0B"
      error: "#EF4444"
      info: "#3B82F6"
  typography:
    font_family: "Inter, system-ui, sans-serif"
    scale: "1.25"
    base_size: 16
  spacing:
    base: 4
    scale: [0, 1, 2, 3, 4, 6, 8, 12, 16, 24, 32]
  radii:
    none: 0
    sm: 4
    md: 8
    lg: 12
    xl: 16
    full: 9999

brands:
  active: "acme"
  palettes:
    acme:
      colors:
        primary: "#1E40AF"
        secondary: "#7C3AED"
        neutral: "slate"
        semantic:
          success: "#16A34A"
          warning: "#CA8A04"
          error: "#DC2626"
          info: "#2563EB"
      typography:
        font_family: "Helvetica Neue, sans-serif"
    partner-co:
      colors:
        primary: "#DC2626"
        secondary: "#F59E0B"
        neutral: "gray"
      typography:
        font_family: "Georgia, serif"

adapters:
  - pencil
  - tailwind
```

### Brand Resolution Logic

- When `brands:` exists and `active` is set, the active palette's values override `tokens.colors` and `tokens.typography.font_family`
- Any key not specified in a brand palette falls back to base `tokens.*`
- Semantic colors can be overridden per brand or fall back to shared semantics
- Spacing and radii are shared across all brands (structural, not identity)
- Configs without `brands:` work exactly as before (backward compatible)

### Brand Scope Per Palette

Each brand can override:
- **Colors:** primary, secondary, neutral, semantic (success, warning, error, info)
- **Typography:** font_family

Shared across all brands (not overridable per brand):
- Spacing (base + scale)
- Radii
- Rules

## Tailwind Adapter — Sync

`/kalos sync` with Tailwind enabled generates two files:

### kalos.tailwind.config.ts

```ts
// Generated by Kalos — do not edit manually
// Regenerate with: /kalos sync
import type { Config } from "tailwindcss";

export const kalosTheme: Partial<Config["theme"]> = {
  extend: {
    colors: {
      primary: "var(--kalos-color-primary)",
      secondary: "var(--kalos-color-secondary)",
      success: "var(--kalos-color-success)",
      warning: "var(--kalos-color-warning)",
      error: "var(--kalos-color-error)",
      info: "var(--kalos-color-info)",
    },
    fontFamily: {
      sans: ["Inter", "system-ui", "sans-serif"],
    },
    spacing: {
      "0": "0px", "1": "4px", "2": "8px", "3": "12px",
      "4": "16px", "6": "24px", "8": "32px",
    },
    borderRadius: {
      none: "0px", sm: "4px", md: "8px", lg: "12px",
      xl: "16px", full: "9999px",
    },
  },
};
```

Tailwind config references CSS custom properties so brand switching works at runtime.

### kalos-tokens.css

```css
/* Generated by Kalos — do not edit manually */
/* Regenerate with: /kalos sync */

:root {
  --kalos-color-primary: #6366F1;
  --kalos-color-secondary: #EC4899;
  --kalos-color-success: #22C55E;
  --kalos-color-warning: #F59E0B;
  --kalos-color-error: #EF4444;
  --kalos-color-info: #3B82F6;
  --kalos-font-family: "Inter", system-ui, sans-serif;
  --kalos-font-base-size: 16px;
  --kalos-spacing-base: 4px;
  --kalos-radius-sm: 4px;
  --kalos-radius-md: 8px;
  --kalos-radius-lg: 12px;
}

/* Multi-brand overrides */
[data-brand="acme"] {
  --kalos-color-primary: #1E40AF;
  --kalos-color-secondary: #7C3AED;
  --kalos-font-family: "Helvetica Neue", sans-serif;
}

[data-brand="partner-co"] {
  --kalos-color-primary: #DC2626;
  --kalos-color-secondary: #F59E0B;
  --kalos-font-family: "Georgia", serif;
}
```

Users import the config into their `tailwind.config.ts` and the CSS file into their global stylesheet. Brand switching at runtime via `data-brand` attribute on a parent element.

## Tailwind Adapter — Validate

`/kalos check` with Tailwind enabled does config-only validation:

1. Look for `tailwind.config.ts` (or `.js`, `.mjs`) in project root
2. Read theme extension values
3. Compare against resolved Kalos tokens:
   - Colors: do config values match Kalos token hex values?
   - Spacing: do values match `base * multipliers`?
   - Radii: do border-radius values match `tokens.radii`?
   - Fonts: does `fontFamily` match `tokens.typography.font_family`?
4. Check for stale generated files — if `kalos.tailwind.config.ts` or `kalos-tokens.css` exist, compare against what sync would generate. Flag if out of date.

Output:
```
Tailwind: tailwind.config.ts
  [OK] Colors match tokens
  [WARN] Spacing value 7px not in token scale
  [WARN] Generated kalos-tokens.css is stale — run /kalos sync
  [OK] Font family matches
```

## /kalos extract — Bootstrap from Existing

New subcommand that reads existing design artifacts and generates `.kalos.yaml`.

### Flow:

1. **Detect available sources** — which adapters have something to read:
   - Pencil: `.pen` files → `search_all_unique_properties` for colors, fonts, spacing, radii
   - Tailwind: `tailwind.config.ts` → parse theme values
   - CSS: `globals.css` or files with `:root` custom properties → parse var definitions

2. **Merge discovered values** — deduplicate colors, identify most-used font as primary, infer spacing base from GCD of spacing values

3. **Present findings:**
   ```
   Kalos Extract — discovered tokens:
     Colors: #6366F1 (12 uses), #EC4899 (8 uses), #22C55E (4 uses)...
     Font: Inter (primary), system-ui (fallback)
     Spacing: base appears to be 4px (values: 4, 8, 12, 16, 24, 32)
     Radii: 0, 4, 8, 12

   Use these as your Kalos config?
   ```

4. **If approved** — write `.kalos.yaml` with discovered values, suggest closest template, run injection

5. **If changes needed** — let user adjust before writing

**Constraint:** Extract only reads from adapter sources (Pencil, Tailwind, CSS). No code parsing, no screenshots — that's the separate code-to-design project.

## Brand Template

New template: `config/templates/brand.yaml`

Strictest of the three:
- Max 6 unique colors (vs 12 modern, 8 minimal)
- Max 1 font family
- WCAG AAA contrast (7.0)
- Ships with example `brands:` section so users see the multi-brand pattern

Includes both adapters (`pencil` + `tailwind`).

## Brand-Aware Behavior

### Onboarding (`/kalos init`)

New questions after existing flow:

8. **Multi-brand?** — "Do you need multiple brand palettes?" → Yes / No
9. **Brand names** — "Name your brands (comma-separated)"
10. **Brand colors + font** — For each brand: primary color, secondary color, font family
11. **Active brand** — "Which brand should be active by default?"

`brands:` section only written if user opts in.

### Brand Switching

User says "switch to partner-co" or `/kalos sync --brand partner-co`:
- Updates `brands.active` in `.kalos.yaml`
- Re-runs instruction injection (CLAUDE.md shows active brand's colors)
- Re-syncs adapters with new active brand

### Instruction Injection

KALOS:START section shows active brand:
```markdown
<!-- KALOS:START - Do not edit this section manually -->
## Design Standards (managed by Kalos)
- Active brand: acme
- Primary: #1E40AF, Secondary: #7C3AED
- Font: Helvetica Neue, sans-serif
...
<!-- KALOS:END -->
```

### Check Validation

- `/kalos check` validates against active brand's resolved colors
- Also validates brand palette consistency: all palettes must define the same keys
- `[ERROR] Brand 'partner-co' missing required key: primary`

### Sync

- Pencil: pushes active brand's colors as main variables
- Tailwind: generates `:root` with active brand defaults + `[data-brand="X"]` blocks for all palettes

## What Ships in v0.2.0

### New
- `/kalos sync` Tailwind adapter — generates `kalos.tailwind.config.ts` + `kalos-tokens.css`
- `/kalos check` Tailwind adapter — config-only validation, stale file detection
- `/kalos extract` — bootstrap `.kalos.yaml` from existing Pencil/Tailwind/CSS artifacts

- Brand-aware onboarding — `/kalos init` asks about multi-brand
- Brand-aware injection — CLAUDE.md shows active brand

- Config schema — `brands:` top-level key with palette structure (colors + typography)
- `/kalos init` — new questions 8-11 for brand setup
- `/kalos check` — runs both Pencil and Tailwind adapters
- `/kalos sync` — runs both adapters
- Templates — `modern` and `minimal` get `tailwind` in adapters, `brand` template added
- Defaults example — updated with `brands:` section and `tailwind` adapter
